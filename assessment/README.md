# 24-Hour Team Assessment — Packet Overview

This packet defines a **fair, comparable** timed team assessment. The design goal is simple: **every team gets the exact same task under the exact same rules**, so their outputs can actually be compared and ranked. Nothing is arbitrary — every scoring criterion maps to a specific ability it is meant to prove.

## What's in this folder

| File | Who it's for | Contents |
|------|--------------|----------|
| `PARTICIPANT_BRIEF.md` | Given to every team | The single task, the rules, timeline, and exactly how to submit. |
| `JUDGING_RUBRIC.md` | Judges (**also published to teams**) | The 100-point rubric, scoring bands, and the judging process. |

> The rubric is shared with participants on purpose. "Criteria clear, upfront" is a core rule — teams should know exactly what they are judged on before they start.

## The design principles (locked)

- **One single task → all teams.** One comparison pool. No per-team variation, because different tasks = incomparable results.
- **24-hour window.** Plenty of time, no artificial pressure. Same clock for everyone.
- **Standardized rules** applied identically to all teams (task, time, team size, allowed tools, submission format).
- **Every criterion proves an ability.** If a criterion doesn't map to a concrete skill, it doesn't belong.
- **A finished product is NOT required.** Teams are judged on demonstrated ability and how far/how soundly they got — not a binary "did it fully work."
- **Ranked hackathon-style** — execution + AI + innovation weighted highest.

## How to run it (organizer checklist)

1. Decide **team size** (default 3–4) and lock it for all teams.
2. Set the **fixed 24h start → end** time; announce the same clock to everyone.
3. Hand out `PARTICIPANT_BRIEF.md` **and** `JUDGING_RUBRIC.md` at kickoff.
4. Assign **≥2 judges per team**; the same judges use the same rubric on every team.
5. Do a **calibration pass** on the first submission before scoring the rest.
6. Collect submissions in the standard format, score to the rubric only, average judge scores, rank.

## Fairness self-check (before you hand it out)

- [ ] One task, all teams, identical submission format → outputs comparable.
- [ ] Every rubric row names the ability it proves.
- [ ] Weights sum to 100; 0/3/5 bands defined.
- [ ] Rules (time, tools, team size) stated once and apply to all.
- [ ] "No output is OK" honored — execution scored on progress/soundness, not completion.
